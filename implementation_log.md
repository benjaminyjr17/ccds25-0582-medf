| 2026-02-24 | Used custom NumPy TOPSIS over pyDecision | We considered integrating pyDecision directly and also a hand-rolled weighted-sum-only baseline for all scoring use-cases. We also reviewed whether to defer TOPSIS entirely and keep deterministic placeholders until late-stage integration. A custom NumPy TOPSIS implementation was selected as the middle path between rigor and dependency overhead. | Implementing TOPSIS in NumPy keeps the mathematical procedure explicit and auditable for assessment. The code exposes normalization, ideal point construction, and closeness computation in one place, which supports academic traceability. It also reduces dependency coupling at the core scoring layer while preserving deterministic behavior under tests. | The deterministic NumPy implementation improves repeatability across environments because algorithmic steps are fully controlled in-project. It limits version-sensitive behavior from external MCDA libraries at the core scoring boundary. This makes it easier to reproduce reported scores from fixed inputs in viva and report contexts. |
| 2026-02-24 | Chose Spearman over Kendall | Kendall's tau was considered because it is robust for ordinal comparisons and has strong interpretability for pairwise concordance. Pearson correlation was also considered but rejected because it assumes interval relationships not guaranteed by ranked dimensions. Spearman's rho was chosen for ranking-based stakeholder disagreement analysis. | Spearman aligns directly with our rank-order conflict framing and is computationally simple for six dimensions. It provides an interpretable continuous signal that maps cleanly to threshold-based conflict levels. The same statistic is also common in applied ethics preference analyses, improving defensibility of metric choice. | Using Spearman yields stable outputs for fixed rankings and avoids sensitivity to absolute score magnitudes. The deterministic rank-vector conversion supports exact replay during testing. This improves reproducibility of conflict reports and threshold classifications. |
| 2026-02-24 | Introduced dual conflict matrices (priority vs salience) | A single conflict matrix based only on contribution-ranked outputs was considered for simplicity. Another option was to expose only weight-priority conflict and omit system-salience effects. We retained both views to separate intrinsic stakeholder priorities from case-specific salience effects. | The weights-only matrix captures structural disagreement between stakeholders independent of any AI system profile. The contribution-based matrix captures disagreement under a specific system's dimension profile, which is useful for scenario interpretation. Presenting both reduces construct ambiguity and improves methodological transparency. | Storing both matrices in metadata preserves backward compatibility while adding explicit analytical context. Each matrix is deterministic for a fixed request, so outputs can be reproduced exactly. This also supports ablation-style explanation in documentation and defense settings. |
| 2026-02-24 | Used salience-weighted L1 distance for Pareto objectives | We considered unweighted L1 distance, Euclidean (L2) distance, and divergence-based alternatives for stakeholder alignment objectives. We also considered optimizing direct utility terms only, but this weakens per-stakeholder fairness interpretation. Salience-weighted L1 was selected to align optimization with case-relevant ethical emphasis. | Salience-weighting ensures dimensions with higher system relevance have proportionally stronger influence on consensus optimization. L1 distance is interpretable, robust, and directly decomposable per dimension and stakeholder. This gives a clear objective statement suitable for academic reporting and examiner challenge. | Objective calculations remain deterministic from request payloads and stakeholder weights. The formulation can be re-evaluated exactly from saved metadata (`x_normalized`, `salience_vector`, objective definition). This improves reproducibility of Pareto ranking and interpretation. |
| 2026-02-24 | Used NSGA-II (pymoo) over brute-force grid search | Brute-force simplex grid search was considered for exhaustive transparency, but dimensionality and resolution tradeoffs make it inefficient for interactive use. Random search was also considered as a fallback-only strategy, but lacks consistent Pareto pressure. NSGA-II provides a principled multi-objective optimizer with practical convergence behavior. | NSGA-II is widely used for Pareto front approximation and supports direct multi-objective optimization without scalarization. It offers a defensible methodology for stakeholder trade-off exploration while remaining configurable by population, generations, and seed. This balance is appropriate for both demonstration and technical evaluation. | Seeding NSGA-II enables repeatable runs under fixed hyperparameters and input payloads. Deduplication and deterministic post-sorting further stabilize returned solution sets. Together, these choices improve reproducibility of reported Pareto candidates for milestone evaluation. |
| 2026-02-24 | Normalized Likert scores via min-max scaling to [0,1] | Alternatives included z-score normalization and direct use of raw 1–5 Likert values. We also considered framework-specific scaling transformations, but this would complicate cross-framework comparability. Min-max normalization to [0,1] was selected as the canonical transformation. | Min-max scaling is simple, bounded, and preserves ordinal direction across unified dimensions. It aligns directly with weighting operations and allows straightforward interpretation of utility-like terms. The mapping is easy to document and verify manually, which is valuable in academic defense. | A fixed normalization formula ensures consistent preprocessing across evaluate, conflict, and pareto flows. This reduces hidden variability and supports deterministic replay from raw inputs. It also simplifies test expectations and reproducible end-to-end validation. |
